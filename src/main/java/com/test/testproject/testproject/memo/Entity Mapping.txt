연관 관계 매핑
    객체와 관계형 데이터베이스 테이블을 서로 매핑하는 것
    1. 방향 :
        - 단방향 , 양방향
        - 두 객체 사이에 하나의 참조용 칠드만 가지고 참고하게 되면 단방향, 두 객체 모두 참조용 필드를 갖고 참조하면 양방향

    2. 다중성 : 일대일, 일대다 , 대대일, 다대다

    3. 연관관계의 주인 :
        -양방향일 때 연관관계에서 관리의 주체가 되는 곳
        - 주인은 이 관계의 주인으로서 외래키를 가지고 테이블을 insert, update 하는 권한을 갖는 것이며 이를 지정하는 것은 JPA에게 누가 주인인지 알려주는 용도
        - mappedBy 속성:
            양방향 매핑에서 관계 미소유 객체가 주인 객체를 가리키기 위해 사용하는 것
            현재 자신의 참조가 참조하는 해당 엔티티에 어떤 변수로 지정되었는지 JPA에게 알려줌
            ex ) @OneToMany(mappedBy = "참조하는 엔티티에 있는 변수명")


    @OneToOne
        1. 단방향 / 일대일
        관계의 주인에게 @JoinColumn(name = 이름)을 붙여준다
        2. 양방향 / 일대일
        주인이 아닌 비소유 객체에 주인객체를 컬럼으로 넣어주면(단방향을 추가) 양방향
        주인이 아닌 객체는 mappedBy 속성을 통해 관계의 주인이 상대편임을 명시
        Ex ) @OneToOne(mappedBy = 주인 객체변수명)

    @ManyToOne
        1. 단방향 / 다대일
        다대일 단방향에서 주인이 다 쪽
        2. 양방향 / 다대일
        주인이 아닌 객체에 @OneToMany를 달아주어 양방향 매핑
        주인이 아닌 객체에 mappedBy 속성을 통해서 관계의 주인이 상대편임을 명시

    @OneToMany
    @ManyToMany
